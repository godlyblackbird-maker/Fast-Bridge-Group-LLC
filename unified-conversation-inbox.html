Sources To Merge

These are the channels and updates that should feed the same contact record.

Gmail

Inbound and outbound threads, approvals, and send status.

Twilio

SMS history, delivery status, replies, and opt-out events.

Call Notes

Manual notes, call outcomes, recordings, and next-step summaries.

Forms

Seller submissions, buyer intake, contact requests, and uploads.

Tasks

Assigned follow-ups, due dates, accountability, and team handoffs.

Property Status

Pipeline changes, comp notes, offer status, and contract milestones.

Why This Matters

The page exists to define the workflow target before you wire the full API stack behind it.

A unified inbox removes channel switching. Instead of checking separate Gmail threads, Twilio threads, call logs, seller forms, and property updates, the team works from one record with one timeline and one next action.

  1. Every user sees the latest message, note, and task without guessing which tool has the newest update.
  2. Admins can audit contact history faster because every activity lands on the same timeline.
  3. Acquisitions and dispositions can hand off a lead without losing context.
  4. Property notes and deal-stage changes stay attached to the same contact thread.
